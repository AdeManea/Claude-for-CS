## Why the move

Multiple SuccessCOACHING products (this one, future webapps, decks, diagrams)
need consistent brand fundamentals. Keeping the theme spec inside one product
project created structural ambiguity — was it "the Customer Success product's
design system" or "the shared SuccessCOACHING design system that happens to
live here?"

Promoting it to `projects/design-system/themes/cowork-artifact/` makes the
relationship explicit: this is one theme among many, sitting alongside the
webapp theme, diagram theme, etc. — all of which inherit from the same core
brand foundations.

## What stayed here

The production artifacts that *implement* the theme — `value-map-artifact.html`
and `success-plan-canvas-artifact.html` — remain in `public/` because they're
products of this project. Each of them now carries a header comment pointing
at the canonical theme spec at the new location.

## What's next

Phase 2 of the migration restructures the canonical design-system project
into explicit `core/` + `themes/` folders, and promotes brand assets
(logo, favicon) up into `core/brand/`. See `projects/design-system/CLAUDE.md`
for status.
